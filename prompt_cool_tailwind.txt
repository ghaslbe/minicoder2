Erstelle eine CRUD-Webanwendung 'Personenverwaltung' mit einem modernen, aufgeraeumten Design. BACKEND in backend/: Flask + SQLite (Datei personen.db), Tabelle person mit Spalten id (autoincrement), name, adresse, telefon. REST-API mit flask-cors: GET /api/persons (alle), POST /api/persons (anlegen), PUT /api/persons/<id> (bearbeiten), DELETE /api/persons/<id> (loeschen). Tabelle beim Start automatisch anlegen. Das Backend MUSS auf Port 5010 laufen (fest, nicht raten oder wechseln). FRONTEND in frontend/: React-App, erstellt mit Vite (https://github.com/vitejs/vite, z.B. via 'npm create vite@latest frontend -- --template react'). Fuer das Styling NICHT Material Web Components oder eine andere Custom-Element-Bibliothek verwenden, sondern Tailwind CSS per CDN-Skript-Tag in frontend/index.html einbinden (<script src="https://cdn.tailwindcss.com"></script>) und ausschliesslich mit Tailwind-Utility-Klassen in den JSX-Dateien gestalten (also KEIN npm-Paket fuer Tailwind, KEINE eigene CSS-Datei fuer Layout-Regeln). Design-Vorgabe: zentrierte Karte (max-width ca. 480-560px) mit weissem Hintergrund, dezentem Schatten und abgerundeten Ecken auf hellgrauem Seitenhintergrund; klare Typografie-Hierarchie (grosse, fette Ueberschrift, ausreichend Zeilenabstand); grosszuegiger Weissraum zwischen den Elementen; ein einheitlicher Akzentton (z.B. Indigo oder Blau) fuer den primaeren Button und Fokus-Zustaende der Eingabefelder; die Personenliste als Karten oder klar getrennte Zeilen mit gut lesbaren Aktions-Buttons (Bearbeiten/Loeschen farblich unterscheidbar). Die App zeigt alle Personen in dieser Liste und erlaubt Anlegen, Bearbeiten und Loeschen ueber ein Formular; spricht das Backend per fetch auf http://localhost:5010 an. Der Vite-Dev-Server MUSS auf Port 8095 laufen (fest, z.B. via 'vite --port 8095' bzw. server.port in vite.config.js). Installiere alle noetigen Abhaengigkeiten (npm install im Frontend, pip install -r requirements.txt im Backend in einer venv) und PRUEFE deine Arbeit wirklich: starte Backend und den Vite-Dev-Server im Hintergrund und teste alle vier REST-Endpunkte per curl, inklusive Fehlerfaellen (PUT/DELETE auf eine nicht existierende ID sollte 404 liefern, nicht stillschweigend Erfolg). Pruefe auch per curl, dass der Vite-Dev-Server auf Port 8095 tatsaechlich HTML ausliefert. Behebe alle Fehler, bevor du finish aufrufst.
